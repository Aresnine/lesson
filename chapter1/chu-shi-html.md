## 1、软件结构的划分

微信、qq、yy、陌陌、内涵段子

```
c/s结构和b/s软件：

1）、c/s软件结构：
    c:client 客户端
    s:server 服务器

    特点：
        1、都有特定的客户端
        2、客户端和服务器都能存储数据
        3、可以通过任意的协议进行通信

2）、b/s软件结构
    b:browser 浏览器
    s:server  服务器

    特点：
        1、所有的b/s结构软件的客户端都是浏览器
        2、所有的数据都存储在服务器，相对安全

b/s结构软件和c/s软件结构的优缺点：
    1、不需要开发特定的客户端（省钱）
    2、所有数据都存储在服务器，服务器压力大,需要花费优化服务器
```

## 2、web发展过程

```
    1、web1.0阶段（静态网站）
         代表：新闻网站、政府门户网站
         特点：
            类似于txt文档提供简单的阅读、下载---》网民不能参与其中
            网站----》人
            网站是信息的提供者，单向性的提供和单一性的理解


    2、web2.0阶段（动态网站----我们正处于的阶段）
         代表：商城、论坛、贴吧
         特点：
            以网络为沟通渠道进行人与人沟通
            人《----》人
            网络是一个平台，用户提供信息，通过网络，其他用户获得。

   3、web3.0阶段（未来阶段）
         特点：
            人工智能、关联数据和语义网络构建，形成人和网络以及网络与人的沟通，同时在seo支持下，提高人与人沟通的便利性
            人《----》网络《-----》人
            web3.0网络成为用户需求理解者和提供者，网络对用户了如指掌，知道用户有什么、要什么以及行为习惯，进行资源筛选、智能匹配，直接给用户答案。
```

## 4、网址url：统一资源定位符

```
www.baidu.com 网址---》不完整的url

完整的url以及各部分的含义

http://www.baidu.com:80/test1/test2/test.php?name=zhangsan&sex=nan#top

1、http:// 超文本传输协议 默认的协议
    两种请求方式：
        get请求：url请求，关于信息的查询、搜索、分页，特点所有内容都会显示在url上
        post请求：向服务器发送请求，要求他接受被附在请求后的实体，并把它当做请求队列中请求url所指定资源的附加新子项。修改密码，大数据传输
2、www.baidu.com 网址
3、：80 ：端口号 默认的端口
4、/test1/test2/test.php 文件的路径
5、？ 分隔符
6、name=zhangsan&sex=nan 所携带的信息
7、#top 片段标识符
```

## 5、HTTP协议

**http协议：超文本传输协议，负责客户端和服务器端的通信**

```
客户端：西门庆
服务器：潘金莲
客户端---http---服务器
```

## 6、HTTP协议的工作流程

```
1、通过访问网络统一资源定位符，我们在浏览器输入网址，请求特定的服务器地址，通过http协议进行交流
    http请求的方式：
        1、get请求
            又叫做url请求，所有的内容都显示在url上，大小有限定
        2、post请求
            向服务器发送指定的请求，请求被附在请求后的实体，大小没有限制有服务器配置文件决定

2、web服务器对于请求的处理
    状态码
        404：找不到网页
        403：请求被拒绝
        200：成功

3、服务器跟客户端断开连接
    关闭客户端，整个流程就结束了。如果我们下次再打开浏览器访问网站的时候，他会重复上面的过程。
    注意：
        http协议是一个无状态的协议，不会记录你的使用、请求状态。
```
